CheckRa1n报错解决方法合集

l1nsn0w 2021-01-02 checkra1n

# Error code: -20 错误

Timed out waiting for bootstrap upload(Likely caused by an incompatible tweak, to enable Safe Mode check the option on main screen) (Error code: -20)

  • -20 错误，官方已经给到解决办法，请在应用程序中勾选 “安全模式（Safe Mode）”选项，重新越狱，看是否解决问题。

# Error code: -31 错误

  • -31 错误虚拟机越狱过程中，容易 -31，重启后也会发生 -31，如果没办法可重新刷一下ios系统，重新越狱。

# Error code: -26 错误

  • -26 错误，-18 错误，请先参照 -20 尝试或参考下面内容。

# Error code: 其他错误

  • 下载最新版本 checkra1n
  • 不要在虚拟机上运行，虚拟机上问题比较多
  • Beta版增加了引导进入 No Substrate Mode 模式的选项。要使用此功能，请在出现苹果徽标时按住调高音量按钮，直到设备完成启动。从那里，您将能够卸载导致问题的所有调整，然后重新启动以返回正常的越狱状态。

# Linux简易版报错解决方法

出现 -77 或者 USB错误，请重启再来，换个usb接口。 出现 -78 请请重启再来，换个usb接口。 出现 -79 请重启再来，换个usb接口。

# 虚拟机

checkra1n -26和-31错误 不支持虚拟机,需要在黑苹果，Ra1nUSB，Linux下越狱

# AMD的黑苹果，错误－31，

AMD的CPU使用checkra1n越狱黑苹果会报错-31

# AMD的Linux简易版越狱，错误－71，

AMD的CPU使用Linux简易版checkra1n越狱会报错-71， 处理器型号 : AMD A10系列，错误－71，ALL DONE 后越狱成功 AMD A8系列，错误－71，ALL DONE 后失败 AMD A7系列，错误－71ALL DONE 后失败

# Checkra1n Linux版 checkn1x

开机出现WELCOME TO GRUB的方法是： 1、电脑开机出现grub， 主要原因是：引导程序丢失所致。 具体原因是：设置BIOS时 ，U盘引导文件损坏 。

2、解决方法是： 重写启动U盘，执行修复属系统引导，修复完成之后即可。

# A7 设备

– iPhone 5S、iPad Air 、iPad Mini 2、 iPad Mini 3 Checkra1n使用Linux越狱错误解决办法： 卡30秒左右进入下一条提示时快速插拔数据线 看图如下： 越狱，显示 setting up the exploit (this is the heap spray)

耐心等待30秒,跳转 not its stuck at "right before trigger"时，快速拔插Usb解接口
